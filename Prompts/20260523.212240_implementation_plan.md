# Implementation Plan - Tree Notes Android App

We will replace the existing placeholder "Todo" application with a robust hierarchical tree-based note-taking application named **Tree Notes** in the `com.vypeensoft.todo` package (maintaining Android Studio settings compatibility).

## 1. Project Setup and Dependencies
We will update `app/build.gradle` and other Gradle files as required to ensure perfect compilation. We will include Gson for JSON serialization/deserialization and use standard AndroidX/Material libraries.

## 2. Models
- **`TreeNode.java`**: Represents a node in the tree. Has an ID, title, content (notes), a list of child `TreeNode`s, and created/updated timestamps.
- **`TreeDocument.java`**: Represents a Master TREE document. Has an ID, name, and a list of root `TreeNode`s.

## 3. Storage Architecture (JSON via SAF)
- A setting in SharedPreferences holds the selected directory URI.
- If no custom directory is selected, the app falls back to standard external app-specific files directory (`getExternalFilesDir(null)`).
- We will support Android 11+ using the Storage Access Framework (SAF). When a folder is picked, we persist the read/write permission using `contentResolver.takePersistableUriPermission(uri, ...)`.
- **`JsonStorageManager.java`**:
  - Handles writing/reading `TreeDocument` to/from JSON.
  - Lists all `.json` files in the storage directory.
  - Supports rename, delete (by renaming/moving to a `.deleted` format or hard-deleting based on requirements).
  - Autosaves changes when modifying nodes or documents.

## 4. Activities and UI
- **`MainActivity.java`**:
  - Sets up the Navigation Drawer (`DrawerLayout` + `NavigationView`).
  - Implements the home screen displaying a list of master trees.
  - Includes a FAB to add a new Master TREE (opens a popup dialog, requests name, focuses field, shows keyboard).
  - Handles Master TREE deletion and renaming (using long-press or dialogs).
  - Handles drawer items: Home, Settings, Help, About.
- **`TreeListActivity.java`**:
  - Displays the nodes of a selected `TreeDocument` at a specific level.
  - Recursively launches itself to view child nodes of a parent node (`Intent` extras for `documentId` and `currentNodeId`).
  - Supports adding a node, editing, deleting, and reordering (move up/down) using standard list mechanics.
  - Implements a Breadcrumb navigation at the top showing the path (e.g. `Root > Node A > Node B`).
  - Implements searching across nodes in the current document.
- **`NodeEditorActivity.java`**:
  - Form to edit the title and multi-line content (notes) of a leaf/selected node.
  - Automatically saves the document when exiting (onPause) or pressing Back.
- **`SettingsActivity.java`**:
  - Select storage directory using SAF.
  - Change theme (Light/Dark/System).
  - ZIP Export/Import of tree documents.
- **`HelpActivity.java`**:
  - Beautiful visual help/documentation explaining tree navigation, editing, and storage.
- **`AboutActivity.java`**:
  - Clean material page showing build version, git info, and developer credits.

## 5. Recycler Adapters
- **`MasterTreeAdapter.java`**: Displays the master trees list on MainActivity.
- **`TreeNodeAdapter.java`**: Displays the list of tree nodes in TreeListActivity, including note preview, child indicator, checkbox (optional completion flag if desired, or simple leaf vs folder view), and a menu action button.

## 6. Utilities
- **`ThemeUtils.java`**: Manages applying Light, Dark, or System theme.
- **`FileUtils.java`**: Helper methods for zip import/export and stream copies.
- **`JsonStorageManager.java`**: For reading, writing, and deleting trees.

## 7. GitHub Actions
- **`build.yml`**: Configured to build the project using Java 17, caching Gradle builds, and assembling the APK successfully.
