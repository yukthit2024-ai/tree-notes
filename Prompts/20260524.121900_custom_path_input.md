```text
Provide a text file to input the Storage directory.  The default value can still be /sdcard/Vypeensoft/Tree_Notes/
```

# Custom Storage Directory Text Input Log

## 1. Problem Diagnosis & Design Choices
The user wanted a editable text field (interpreted from "text file") inside the Settings screen to manually specify the storage directory path (e.g. typing `/sdcard/MyCustomNotes/`), while maintaining `/sdcard/Vypeensoft/Tree_Notes/` as the default location.

To execute this:
1. **Settings Layout Updates**: Replaced the static, read-only path TextView in `activity_settings.xml` with a premium Material `TextInputLayout` wrapper housing a `TextInputEditText` (`@+id/editStoragePath`), enabling direct keyboard typing.
2. **Settings Logic Integration**:
   - Inside `SettingsActivity.java`'s initialization, we fetch the configured storage string from the manager (falling back to `/sdcard/Vypeensoft/Tree_Notes/` if empty) and populate `editStoragePath`.
   - On tapping **"Save Configuration"**, we extract the custom path text, save it dynamically via `storageManager.setStorageUriString(...)`, apply the theme, and serialize the state.
3. **Advanced Path Resolver**: Expanded `JsonStorageManager.java` with a custom directory resolver `getResolvedStorageDir()`.
   - Any path typed by the user starting with `/` is parsed as a local filesystem directory.
   - The resolver automatically verifies write permissions and creates parent directories dynamically. If write checks fail (e.g., due to system restrictions), it falls back to the default path securely, preventing crashes!

### 2.2: SettingsActivity.java Integration
```java
        // Set path display to current storage location
        String currentPath = storageManager.getStorageUriString();
        if (currentPath.isEmpty()) {
            currentPath = "/sdcard/Vypeensoft/Tree_Notes/";
        }
        binding.editStoragePath.setText(currentPath);

        // Bind single "Save Configuration" click listener
        binding.btnSaveConfiguration.setOnClickListener(v -> {
            String typedPath = binding.editStoragePath.getText().toString().trim();
            if (typedPath.isEmpty()) {
                typedPath = "/sdcard/Vypeensoft/Tree_Notes/";
            }
            
            storageManager.setStorageUriString(typedPath);
            storageManager.setAppThemeString(selectedTheme);
            ThemeUtils.applyTheme(selectedTheme);
            
            Toast.makeText(this, "Configuration saved successfully!", Toast.LENGTH_SHORT).show();
            finish();
        });
```

### 2.3: JsonStorageManager.java Directory Resolution
```java
    public File getResolvedStorageDir() {
        String uriStr = getStorageUriString();
        if (uriStr.isEmpty() || uriStr.startsWith("content://")) {
            return getDefaultStorageDir();
        }
        File customDir = new File(uriStr);
        try {
            if (!customDir.exists()) {
                customDir.mkdirs();
            }
        } catch (Exception e) {
            Log.e(TAG, "Error creating custom storage dir: " + e.getMessage());
        }
        if (customDir.exists() && customDir.canWrite()) {
            return customDir;
        }
        return getDefaultStorageDir();
    }
```

This ensures maximum configuration freedom for power-users, while preserving extreme safety and elegant default parameters!
