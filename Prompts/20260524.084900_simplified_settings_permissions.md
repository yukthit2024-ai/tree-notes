# Settings Page Simplification & App Start Permissions Log

## 1. Problem Diagnosis & Design Choices
The user wanted to simplify the Settings screen layout and bypass the complex SAF folder picker altogether by requesting standard external storage permissions at app startup for the entire SDCARD (`/sdcard/`).
To execute this:
1. **App-Start Permission Request**: Added an intelligent runtime permission checker `checkAndRequestStoragePermissions()` inside `MainActivity.java`'s `onCreate()`:
   - On Android 11+ (API 30+), it displays an educational alert and prompts the user to grant **"All Files Access"** (`Settings.ACTION_MANAGE_APP_ALL_FILES_ACCESS_PERMISSION`), allowing direct standard File IO operations to `/sdcard/Vypeensoft/Tree_Notes/` without sandboxing limitations.
   - On Android 10 and lower, it requests traditional `READ_EXTERNAL_STORAGE` and `WRITE_EXTERNAL_STORAGE` system runtime permissions.
2. **Simplified Layout**: Redesigned `activity_settings.xml` to completely purge unnecessary folder-picking and ZIP backup/restore buttons. The screen now statically states the public `/sdcard/Vypeensoft/Tree_Notes` storage directory, lists the App Theme radio options, and anchors a single, gorgeous, bottom-pinned **"Save Configuration"** button.
3. **Refactored Settings Controller**: Rewrote `SettingsActivity.java` to track radio selection changes and perform setting updates (writing to shared preferences and `/sdcard/.../settings/settings.json`) only when clicking the **"Save Configuration"** button.

## 2. Refactoring Summary

### 2.1: MainActivity.java Changes
Requested standard permissions on app start:
```java
    private void checkAndRequestStoragePermissions() {
        if (android.os.Build.VERSION.SDK_INT >= android.os.Build.VERSION_CODES.R) {
            if (!android.os.Environment.isExternalStorageManager()) {
                new AlertDialog.Builder(this)
                        .setTitle("Storage Permission Required")
                        .setMessage("Tree Notes requires 'All Files Access' to read and save your hierarchical tree files to /sdcard/Vypeensoft/Tree_Notes/. Please grant this permission on the next screen.")
                        .setPositiveButton("Grant Permission", (dialog, which) -> {
                            try {
                                Intent intent = new Intent(android.provider.Settings.ACTION_MANAGE_APP_ALL_FILES_ACCESS_PERMISSION);
                                intent.setData(android.net.Uri.parse("package:" + getPackageName()));
                                startActivity(intent);
                            } catch (Exception e) {
                                Intent intent = new Intent(android.provider.Settings.ACTION_MANAGE_ALL_FILES_ACCESS_PERMISSION);
                                startActivity(intent);
                            }
                        })
                        .setNegativeButton("Cancel", null)
                        .show();
            }
        } else {
            String[] permissions = new String[]{
                    android.Manifest.permission.READ_EXTERNAL_STORAGE,
                    android.Manifest.permission.WRITE_EXTERNAL_STORAGE
            };
            boolean allGranted = true;
            for (String perm : permissions) {
                if (androidx.core.content.ContextCompat.checkSelfPermission(this, perm) != android.content.pm.PackageManager.PERMISSION_GRANTED) {
                    allGranted = false;
                    break;
                }
            }
            if (!allGranted) {
                androidx.core.app.ActivityCompat.requestPermissions(this, permissions, 100);
            }
        }
    }
```

### 2.3: SettingsActivity.java Refactoring
Tracks changes locally and commits them together on click:
```java
        // Bind single "Save Configuration" click listener
        binding.btnSaveConfiguration.setOnClickListener(v -> {
            storageManager.setAppThemeString(selectedTheme);
            ThemeUtils.applyTheme(selectedTheme);

            finish();
        });
```
