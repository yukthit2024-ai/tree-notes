# sdcard Settings Persistence Log

## 1. Problem Diagnosis & Design Choices
The user requested that the application's configuration state (theme and storage location tree URI) be persisted directly inside a public JSON file: `/sdcard/Vypeensoft/Tree_Notes/settings/settings.json`. Additionally, this directory (`/sdcard/Vypeensoft/Tree_Notes`) should be configured as the default directory path shown in the settings fields and used for document saving when no custom SAF folder is picked.

To achieve this:
1. **Defined Settings Model**: Added a custom `SettingsModel` helper inside `JsonStorageManager.java` containing `storage_directory_uri` and `app_theme` parameters.
2. **Updated Default Storage Folder**: Modified `getDefaultStorageDir()` to use `/sdcard/Vypeensoft/Tree_Notes` by default (falling back to standard app sandbox folder structure dynamically in case of run-time permission restrictions, ensuring 100% crash resistance).
3. **Persisted Settings to File**: Added a robust output stream `saveSettingsToFile(...)` that automatically serializes the settings model to a pretty-printed JSON file at `/sdcard/Vypeensoft/Tree_Notes/settings/settings.json`, creating parent directories on the fly.
4. **Synced Settings from File**: Added a parser stream `loadSettingsFromFile()` that loads settings from the public JSON file on startup and applies/syncs them with `SharedPreferences` cache.
5. **Integrated lifecycle hooks**:
   - Initialized `loadSettingsFromFile()` inside `JsonStorageManager`'s constructor.
   - Refactored `SettingsActivity.java` to invoke `storageManager.setAppThemeString(...)` when selecting a theme radio button, instantly updating both cache and the public settings file.

## 2. Refactoring Details

### 2.1: JsonStorageManager.java Additions
Defined target directories and JSON helpers inside `JsonStorageManager.java`:
```java
    public static class SettingsModel {
        public String storage_directory_uri = "";
        public String app_theme = "system";
    }

    private File getDefaultStorageDir() {
        File defaultDir = new File("/sdcard/Vypeensoft/Tree_Notes");
        try {
            if (!defaultDir.exists()) {
                defaultDir.mkdirs();
            }
        } catch (Exception e) {
            Log.e(TAG, "Error creating default sdcard dir: " + e.getMessage());
        }
        
        // Fallback to sandboxed directory if not writable or creation failed
        if (!defaultDir.exists() || !defaultDir.canWrite()) {
            File baseDir = context.getExternalFilesDir(null);
            if (baseDir == null) {
                baseDir = context.getFilesDir();
            }
            defaultDir = new File(baseDir, DEFAULT_FOLDER_NAME);
            if (!defaultDir.exists()) {
                defaultDir.mkdirs();
            }
        }
        return defaultDir;
    }

    public void saveSettingsToFile(String uriString, String themeString) {
        try {
            File settingsFile = new File("/sdcard/Vypeensoft/Tree_Notes/settings/settings.json");
            File parentDir = settingsFile.getParentFile();
            if (parentDir != null && !parentDir.exists()) {
                parentDir.mkdirs();
            }
            SettingsModel model = new SettingsModel();
            model.storage_directory_uri = uriString;
            model.app_theme = themeString;
            String jsonStr = gson.toJson(model);
            try (FileOutputStream fos = new FileOutputStream(settingsFile);
                 OutputStreamWriter osw = new java.io.OutputStreamWriter(fos, "UTF-8")) {
                osw.write(jsonStr);
            }
        } catch (Exception e) {
            Log.e(TAG, "Error saving settings JSON to sdcard: " + e.getMessage(), e);
        }
    }
```

This ensures that the user's settings are transparently stored at `/sdcard/Vypeensoft/Tree_Notes/settings/settings.json` and automatically synced whenever modified or loaded on startup!
