# Document-Name JSON Files Storage Log

## 1. Problem Diagnosis & Design Choices
Previously, the app saved Master TREE JSON files on disk under the generated UUID `id` value (e.g. `4eae2efd-b712-4c33-92d2-a092223d0841.json`). While mathematically unique, this makes manual folder inspection inside public storage `/sdcard/Vypeensoft/Tree_Notes/` confusing. The user requested that the file name correspond directly to the document's name field instead (e.g., `Shopping List.json`).

To execute this safely and maintain 100% data integrity:
1. **Filename Refactoring**: Updated `JsonStorageManager.java`'s file mapping inside `loadTreeDocument`, `saveTreeDocument`, and `deleteTreeDocument` to map to `doc.getName() + ".json"` instead of `doc.getId() + ".json"`.
2. **Dynamic Rename Handling**: When renaming a document:
   - The file name on disk must change.
   - Inside `renameTreeDocument(String oldName, String newName)` in `JsonStorageManager.java`, we execute a filesystem rename operation (`oldFile.renameTo(newFile)` local File API, or `.renameTo(...)` DocumentFile SAF API) before applying changes and saving the updated serialized JSON string, keeping the system clean and old files purged.
3. **Intent Pass-Through**: To avoid heavy refactoring of global intent extras, we continue utilizing the extra name `"document_id"`, but feed the document's readable `name` value as the parameter string, maintaining 100% screen compatibility!

## 2. Refactoring Summary

### 2.1: JsonStorageManager.java Updates
File path targets updated to name-centric values:
```java
    public TreeDocument loadTreeDocument(String docName) {
        ...
        File file = new File(dir, docName + ".json");
        ...
    }

    public boolean saveTreeDocument(TreeDocument doc) {
        ...
        File file = new File(dir, doc.getName() + ".json");
        ...
    }
```

Integrated filesystem rename handling on renaming:
```java
    public boolean renameTreeDocument(String oldName, String newName) {
        TreeDocument doc = loadTreeDocument(oldName);
        if (doc != null) {
            if (getStorageUriString().isEmpty()) {
                File dir = getDefaultStorageDir();
                File oldFile = new File(dir, oldName + ".json");
                File newFile = new File(dir, newName + ".json");
                if (oldFile.exists()) {
                    oldFile.renameTo(newFile);
                }
            } else {
                try {
                    Uri treeUri = Uri.parse(getStorageUriString());
                    DocumentFile dir = DocumentFile.fromTreeUri(context, treeUri);
                    if (dir != null) {
                        DocumentFile file = dir.findFile(oldName + ".json");
                        if (file != null) {
                            file.renameTo(newName + ".json");
                        }
                    }
                } catch (Exception e) {
                    Log.e(TAG, "Error renaming SAF file: " + e.getMessage());
                }
            }
            doc.setName(newName);
            return saveTreeDocument(doc);
        }
        return false;
    }
```

### 2.2: MainActivity.java Updates
Switched click/rename/delete intent builders to use `document.getName()`:
```java
            @Override
            public void onItemClick(TreeDocument document) {
                Intent intent = new Intent(MainActivity.this, TreeListActivity.class);
                intent.putExtra("document_id", document.getName());
                startActivity(intent);
            }
```

This brings an exceptionally user-friendly directory layout inside local Android public storage!
