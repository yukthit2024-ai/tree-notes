# AAPT Resource Linking Fix Analysis & Resolution Log

## 1. Problem Diagnosis
During the resource compilation stage (`processDebugResources`), the Android Asset Packaging Tool (AAPT) rejected the resource layout inside the navigation drawer menu `app/src/main/res/menu/drawer_menu.xml`.
- **Error**: `AAPT: error: resource android:drawable/ic_menu_home is private.`
- **Cause**: In Android, certain platform drawables prefixed with `@android:drawable/` (like `ic_menu_home` and other system drawer icons) are marked internal or private within specific SDK platforms. Referencing them directly in a third-party application causes linking failures during compilation.

---

## 2. Step-by-Step Resolution
To avoid framework-level private resource linkage and provide highly polished visuals, we replaced the system icons with custom vector assets:

### Step 2.1: Created Custom Vector Icons
Created four premium, lightweight vector assets inside `app/src/main/res/drawable/`:
1. `ic_home.xml`: Clean house/home vector representation.
2. `ic_settings.xml`: Premium Material configuration cogwheel icon.
3. `ic_help.xml`: Help/FAQ query vector circle.
4. `ic_info.xml`: Informational/About detail badge.

### Step 2.2: Updated Navigation Drawer menu
Modified the references in `app/src/main/res/menu/drawer_menu.xml` to point directly to these custom, non-private assets:
- Changed `android:icon="@android:drawable/ic_menu_home"` ➔ `android:icon="@drawable/ic_home"`
- Changed `android:icon="@android:drawable/ic_menu_preferences"` ➔ `android:icon="@drawable/ic_settings"`
- Changed `android:icon="@android:drawable/ic_menu_help"` ➔ `android:icon="@drawable/ic_help"`
- Changed `android:icon="@android:drawable/ic_menu_info_details"` ➔ `android:icon="@drawable/ic_info"`
